  <action>
1. Create `src/app/api/settings/route.ts` — POST endpoint:
   - Auth check: createSupabaseServer() → getUser() → 401 if no user
   - Parse JSON body, extract `name` field
   - Validate: name must be a non-empty string, trim it, max 50 chars
   - Use admin client: `admin.from("fans").update({ name: name.trim() }).eq("email", user.email)`
   - If the fan has no fans row yet (update affects 0 rows), upsert: create a fans row with email and name
   - Return { success: true } on success, { error: string } on failure

2. Create `src/app/settings/page.tsx` as a server component:
   - Auth check: createSupabaseServer() → getUser() → redirect /auth/login if no user
   - Fetch fan data: admin.from("fans").select("id, email, name").eq("email", user.email).single()
   - Pass fan (or null) and user email to SettingsClient

3. Create `src/app/settings/settings-client.tsx` as a "use client" component:
   - Import useRouter from next/navigation, useState
   - Import createSupabaseBrowser from @/lib/supabase
   - Import icons from lucide-react (ArrowLeft, Save, LogOut)
   - Import toast from sonner
   - Import Link from next/link

   **Layout:** Dark bg-bg full page, max-w-md centered, consistent with profile page

   **Header:** "Settings" title, back link to /profile (ArrowLeft icon)

   **Display Name Section:**
   - Label: "Display Name"
   - Text input (dark styled: bg-bg-card, border-light-gray/20, text-white, focus:border-pink)
   - Pre-filled with fan.name or empty string
   - Save button (gradient bg from pink to purple, like other CTAs)
   - On save: fetch POST /api/settings with { name }, show toast on success/error
   - If fan is null (no fans row): show input anyway — the API will upsert

   **Account Section:**
   - Show user email (read-only, text-gray)
   - Logout button: red-ish styling (text-red-400 or similar), calls createSupabaseBrowser().auth.signOut() then router.push("/")
   - Show toast "Signed out" on logout

   **Aesthetic:** bg-bg background, bg-bg-card for form sections, brand accent colors, Poppins font (already global), consistent with profile and dashboard pages.
