  <action>
Create a server component at `src/app/leaderboard/page.tsx` that:

1. Uses `createSupabaseAdmin()` to query leaderboard data (bypass RLS for public rankings).
2. Optionally uses `createSupabaseServer()` to get the current user session (for "your position" highlight). If no session, pass `currentFanId: null`.

3. **Fan leaderboard query** — For each time period (weekly = last 7 days, monthly = last 30 days, all-time = no date filter), query `collections` table grouped by `fan_id`, counting DISTINCT `performer_id` as the ranking metric. Join `fans` table for display_name/email. Order by count DESC, limit 10. Use `created_at` for time filtering.

4. **Performer leaderboard query** — Query `collections` table grouped by `performer_id`, counting DISTINCT `fan_id`. Join `performers` table for name, slug, photo_url. Order by count DESC, limit 10. Same time filters.

5. Structure the data as three objects (weekly, monthly, allTime), each containing `{ fans: FanEntry[], performers: PerformerEntry[] }` where:
   - `FanEntry = { rank: number, fanId: string, name: string, count: number, topTier: string }`
   - `PerformerEntry = { rank: number, performerId: string, name: string, slug: string, photoUrl: string | null, fanCount: number }`

6. For the logged-in user: look up their fan record by email from the `fans` table, pass their `fanId` to the client component.

7. Pass all data + `currentFanId` to `LeaderboardClient`.

Add OG metadata: title "Leaderboard | DECIBEL", description "Top fans and performers in the Chicago underground scene".
