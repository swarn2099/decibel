  <files>scripts/content/generate-week.ts, scripts/content/spotlight.ts, scripts/content/roundup.ts, scripts/content/teaser.ts</files>
  <action>
Update `generate-week.ts` to:

1. **Dated output directory**: Create a dated subdirectory under `content/output/` for each batch run. Format: `content/output/week-YYYY-MM-DD/`. Use `mkdirSync` with `recursive: true`.

2. **Update individual generators' output paths**: The current spotlight/roundup/teaser scripts hardcode `content/output/` as their output path. Rather than modifying each generator's signature, have `generate-week.ts` pass the output directory to each generator. Two approaches:
   - **Option A (preferred)**: Update `generateSpotlight`, `generateRoundup`, `generateTeaser` to accept an optional `outputDir` parameter. If provided, use that instead of `content/output/`. Default to `content/output/` for standalone usage.
   - **Option B**: Let them output to `content/output/` and then move files into the weekly directory.

   Go with Option A — cleaner, avoids file move race conditions.

3. **Fix the post mix**: Currently the teaser count is `Math.floor(Math.random() * 2) + 1` (1-2). The spotlight count is `Math.min(3, shuffled.length)` (up to 3). This gives 3 + 1 + 1-2 = 5-6, or with 2 spotlights = 4-5. Ensure minimum 5 posts total:
   - Spotlights: Pick exactly `Math.min(3, available)` (target 2-3)
   - Roundup: Always 1
   - Teasers: Pick enough to reach at least 5 total: `Math.max(1, 5 - spotlightCount - 1)` capped at 2
   - This guarantees 5-7 posts

4. **Print summary**: At the end, print the output directory path and a count of generated files by type.

5. **Update spotlight.ts, roundup.ts, teaser.ts**: Add optional `outputDir?: string` parameter to each exported function. When provided, use `resolve(outputDir, filename)` instead of `resolve(process.cwd(), 'content/output/filename')`. Keep standalone CLI usage unchanged (defaults to `content/output/`).
