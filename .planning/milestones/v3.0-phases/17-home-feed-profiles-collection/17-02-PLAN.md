Database schema (performer columns):
- id, name, slug, bio, photo_url, soundcloud_url, spotify_url, spotify_id,
  mixcloud_url, ra_url, instagram_handle, city, genres (text[]),
  follower_count, claimed, is_chicago_resident, created_at

Events table: id, event_date, start_time, end_time, external_url, performer_id, venue_id
Venues table: id, name, slug, address, lat, lng
Collections table: fan_id, performer_id, verified, capture_method, event_date
Fan_tiers table: fan_id, performer_id, scan_count, current_tier
Founder_badges table: fan_id, performer_id, awarded_at

Web app artist page data flow (reference):
- getPerformer(slug): performers table, select * where slug=slug
- getUpcomingEvents(performerId): events joined with venues, event_date >= today, limit 6
- getFanCount(performerId): count of collections where performer_id
- getFounder(performerId): founder_badges joined with fans, select awarded_at + fan name/avatar
- getSimilarArtists(performerId, genres): performers where genres overlap, neq id, limit 8

Web app artist profile has existing API routes:

- `useArtistEvents(performerId: string | undefined)`: Query events joined with venues (select id, event_date, start_time, end_time, external_url, venue:venues(name, slug, address)). - `useArtistFanCount(performerId: string | undefined)`: Query collections count where performer_id=performerId. Return number. staleTime: 5 min. enabled: !!performerId.

- `useArtistFounder(performerId: string | undefined)`: Query founder_badges where performer_id, join fans table (select name, email, avatar_url). Return { name, avatar_url, awarded_at } or null. Handle the join unwrapping. staleTime: 10 min. enabled: !!performerId.

- `useSimilarArtists(performerId: string | undefined, genres: string[] | null)`: Query performers where genres overlaps with provided genres, neq id, order by follower_count desc nulls last, limit 8. Return array of { id, name, slug, photo_url, genres }. staleTime: 10 min. enabled: !!performerId && !!genres?.length.
