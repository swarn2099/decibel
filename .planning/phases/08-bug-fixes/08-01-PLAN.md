Fix 4 bugs in the artist profile page:

**BUG-01 — Instagram double-URL:**
The `socialLinks` array constructs Instagram href as `https://instagram.com/${performer.instagram_handle}`. But `instagram_handle` in the DB may already be a full URL (e.g., `https://www.instagram.com/djname`). Add a helper function `cleanInstagramHandle(handle: string): string` that:
- If handle starts with `http` or contains `instagram.com`, extract the username from the path
- Strip leading `@` if present
- Return the clean username
Use this in the socialLinks construction so the href is always `https://instagram.com/{clean_username}` and the label is always `@{clean_username}`.

**BUG-02 — Fan count display:**
Add a Supabase query function `getFanCount(performerId: string): Promise<number>` that counts distinct `fan_id` rows in `collections` where `performer_id` matches. Call it alongside `getUpcomingEvents`. In the hero meta section (the flex div with follower_count and city), add a fan count display:
- If fanCount > 0: show `{fanCount} fans` with the Users icon (already imported)
- If fanCount === 0: hide entirely (do not render)
Never show a bare "0" number.

**BUG-03 — Hide empty sections:**
The "Tracks" section currently shows whenever `performer.soundcloud_url` is truthy, but the SoundCloud embed may load an empty widget. This is acceptable behavior since the URL existing means there should be content. However, ensure the "Upcoming Shows" placeholder section (lines 393-406, the "No upcoming shows listed yet" dashed-border card) is removed entirely — an empty state placeholder for shows is not useful content and clutters the page. Only show the "Upcoming Shows" section when there are actual events.

**BUG-06 — Dominant Collect CTA:**
The current Collect button (lines 262-268) is a small `rounded-full` pill with gradient. Make it visually dominant:
- Change to `w-full sm:w-auto` so it's full-width on mobile
- Increase padding: `px-10 py-4` (from px-7 py-3)
- Increase font size: `text-base font-extrabold` (from text-sm font-bold)
- Add a glow effect: `shadow-[0_0_20px_rgba(255,77,106,0.4)]`
- Keep the gradient and hover:scale-105
- Move it above the social link pills by wrapping in a div that stacks on mobile: on mobile the button should be full-width above the social icons, on desktop it stays inline
