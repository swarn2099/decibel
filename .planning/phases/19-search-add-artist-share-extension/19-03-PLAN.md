<!-- Expo Linking and deep link patterns -->

Expo Router handles deep links via the app's URL scheme and universal links.
- Custom scheme: `decibel://` (configured in app.json)
- Deep link format: `decibel://shared?url=<encoded_url>`

For Android intent filters, app.json needs:
```json
{
  "android": {
    "intentFilters": [{
      "action": "android.intent.action.SEND",
      "category": ["android.intent.category.DEFAULT"],
      "data": [{ "mimeType": "text/plain" }]
    }]
  }
}
```

For iOS share extension: Expo doesn't have built-in share extension support.
The pragmatic approach for v3.0 is deep linking via custom URL scheme:
- `decibel://shared?url=<encoded_url>`
- This works when another app opens a URL with the decibel:// scheme

For actual iOS Share Sheet integration (appearing in the share menu), this requires a native iOS extension which is out of scope for v3.0 Expo managed workflow. Instead, implement:
1. Deep link handler for `decibel://shared?url=<url>`
2. Android intent filter for receiving shared text (URLs)
3. In-app "Paste a link" option on the search screen as fallback

Create `/home/swarn/decibel-mobile/src/lib/urlParser.ts` with:

**parseArtistUrl(url: string)** — returns `{ platform, artistId, artistName? }` or `null`:

Supported URL patterns:
- Spotify: `https://open.spotify.com/artist/{id}` or `spotify:artist:{id}` — extract `id`, platform = "spotify"
- SoundCloud: `https://soundcloud.com/{username}` — extract `username`, platform = "soundcloud"
- Instagram: `https://instagram.com/{handle}` or `https://www.instagram.com/{handle}` — extract `handle`, platform = "instagram"

Return type:
```typescript
export type ParsedArtistUrl = {
  platform: "spotify" | "soundcloud" | "instagram";
  artistId: string; // Spotify ID, SoundCloud username, or Instagram handle
};

export function parseArtistUrl(url: string): ParsedArtistUrl | null;
```

Handle edge cases:
- URLs with query params or trailing slashes (strip them)
- URLs with tracking params (utm_, si=, etc.)
- Both http and https
- Both www and non-www
- Return null for unrecognized URLs

Also export:
```typescript
export function extractUrlFromSharedText(text: string): string | null;
```
This extracts the first URL from shared text (Android shares text/plain which may include extra text around the URL). Use a simple URL regex to find the first https:// URL in the text.
