Create `/home/swarn/decibel-mobile/src/hooks/useSearch.ts` with two TanStack Query hooks:

**useDecibelSearch(query: string)**:
- Uses TanStack `useQuery` with queryKey `["decibelSearch", query]`
- Only enabled when query.length >= 2
- Queries Supabase directly: `supabase.from("performers").select("id, name, slug, photo_url, genres, follower_count").ilike("name", `%${query}%`).limit(10)`
- Returns array of Performer (partial, matching the select fields)
- staleTime: 30 seconds (searches are ephemeral)

**useSpotifySearch(query: string, enabled: boolean)**:
- Uses TanStack `useQuery` with queryKey `["spotifySearch", query]`
- Only enabled when `enabled` is true AND query.length >= 2
- Calls web API: `fetch("https://decibel-three.vercel.app/api/spotify/search?q=${encodeURIComponent(query)}")`
- Returns `{ existing: DecibelResult[], results: SpotifyResult[], spotify_error?: string }`
- staleTime: 60 seconds

**Types to export from this file:**
```typescript
export type SpotifyArtistResult = {
  id: string;
  name: string;
  photo_url: string | null;
  spotify_url: string;
  genres: string[];
  followers: number;
  monthly_listeners?: number;
};
```

Both hooks should NOT debounce internally — debouncing happens in the search screen via a `debouncedQuery` state (300ms delay). This keeps hooks pure.

Use the existing `supabase` client from `@/lib/supabase`.
