  <action>
Create `scripts/scrapers/bandsintown.ts` that scrapes Chicago electronic music events from Bandsintown's public API.

**Approach — venue-based event lookup:**
Bandsintown has a public endpoint for venue events: `https://rest.bandsintown.com/v4/venues/{venue_id}/events?app_id=decibel`
The `app_id` parameter just needs any non-empty string (it's a free API, no key needed).

Also use the artist events endpoint for known Chicago artists: `https://rest.bandsintown.com/artists/{artist_name}/events?app_id=decibel&date=upcoming`

**Implementation steps:**

1. Define a list of target Chicago venue IDs/names to query. Start with venue search: `https://rest.bandsintown.com/v4/venues/search?query=chicago&app_id=decibel` to discover venue IDs, then hardcode the electronic music ones. Key venues to target: Smartbar, Spybar, Soundbar, Concord Music Hall, Radius, Prysm, The Mid, Podlasie, Primary, Metro, Empty Bottle, Sleeping Village, Smart Bar.

2. For each venue, fetch upcoming events via the venue events endpoint.

3. Extract artist names, event dates, and venue info from each event response.

4. Cross-reference with existing DB performers using the same pattern as ra.ts:
   - Pre-fetch all existing performers
   - Match by normalized name key, then by slug, then by fuzzy `namesMatch`
   - If existing: update photo/social links if we have new data
   - If new: insert with `source: "bandsintown"` conventions

5. Insert events with proper venue linkage (use same `matchVenueSlug` + create-if-missing pattern from ra.ts).

6. Use `isNonArtistName` to filter out event-name entries.

7. Export `scrapeBandsintown` function for use in run-all.ts.

8. Rate limit: 200ms between API calls (Bandsintown is generous but be polite).

9. Log stats at end: events fetched, unique artists, new performers, updated performers, events inserted.

**API response shape (events endpoint):**
```json
{
  "id": "123",
  "artist_id": "456",
  "datetime": "2024-01-15T20:00:00",
  "venue": { "name": "Smartbar", "city": "Chicago", "region": "IL", "country": "US", "latitude": "41.9", "longitude": "-87.6" },
  "lineup": ["DJ Name 1", "DJ Name 2"],
  "offers": [{ "url": "https://..." }],
  "artist": { "name": "...", "image_url": "...", "thumb_url": "...", "facebook_page_url": "..." }
}
```

**Note:** If the venue search API doesn't work as expected (Bandsintown changes endpoints), fall back to the artist-search approach: query events for known Chicago-based performers from our DB who don't have Bandsintown data yet, and discover new venues/co-performers through their event lineups.
