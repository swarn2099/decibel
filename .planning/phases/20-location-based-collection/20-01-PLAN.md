3. Create `src/hooks/useLocation.ts`:
   - Export `useLocation()` hook that wraps expo-location
   - On mount, check `Location.getForegroundPermissionsAsync()` to get current status (don't request yet)
   - Expose `permissionStatus: 'granted' | 'denied' | 'undetermined'`
   - Expose `requestPermission()` async function that calls `Location.requestForegroundPermissionsAsync()` — this is "While Using" only, NOT `requestBackgroundPermissionsAsync`

4. Create `src/hooks/useVenueDetection.ts`:
   - Export `useVenueDetection()` hook
   - Accepts optional `{ enabled?: boolean }` param (default true)
   - Uses `useLocation()` internally
   - When enabled and permission granted, uses TanStack Query (`useQuery`) to:
     a. Get current position via `getCurrentPosition()`
     b. Query Supabase: `supabase.from('venues').select('id, name, slug, address, lat, lng, geofence_radius')` — fetch all venues (there are only Chicago venues, small set)
     c. For each venue with lat/lng, compute haversine distance from fan's current position
     d. Filter to venues within their geofence_radius (default 200m if null)
     e. For matching venues, query events: `supabase.from('events').select('id, event_date, performer_id, performers(id, name, slug, photo_url)').eq('venue_id', venueId)` where event_date is today and is_live is true (or event_date matches today's date)
     f. Return `ActiveVenueEvent[]` — each venue with its performing artists
   - Haversine distance function: implement inline (simple ~10 line function, no library needed). Formula: `a = sin²(Δlat/2) + cos(lat1) * cos(lat2) * sin²(Δlng/2)`, `d = 2R * atan2(√a, √(1-a))` where R = 6371000 meters
   - Query key: `['venueDetection', lat, lng]` (rounded to 4 decimal places to avoid constant refetching)
   - staleTime: 2 minutes (don't spam location checks)
   - enabled: only when `hasPermission && enabled`
   - If permission denied, return empty array (graceful degradation — LOC-07)
   - Expose: `nearbyEvents: ActiveVenueEvent[]`, `isChecking: boolean`, `refetch()`

```typescript
interface LocationState {
  // Track which events the fan dismissed (tapped X on banner)
  dismissedEventIds: string[];
  dismissEvent: (eventId: string) => void;
  isEventDismissed: (eventId: string) => boolean;
  clearDismissed: () => void; // called daily or on app restart

  // Track if we've shown the permission explanation
  hasShownExplanation: boolean;
  setExplanationShown: () => void;

  // Track if location permission was explicitly denied (for "I'm at a show" fallback messaging)
  permissionDenied: boolean;
  setPermissionDenied: (denied: boolean) => void;
}
```

- `dismissedEventIds` persisted to MMKV as JSON string
- `dismissEvent` adds eventId to array and persists
- `isEventDismissed` checks array membership
- `clearDismissed` resets the array (call when today's date changes)
- `hasShownExplanation` persisted to MMKV boolean
- `permissionDenied` persisted to MMKV boolean

This store lets the banner system remember which events the fan already said no to, and tracks whether we've shown the explanation screen before the OS permission dialog.
