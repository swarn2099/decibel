  <action>
    1. Create src/lib/pushNotifications.ts — server-side helper for Expo Push API:
       - `sendPushNotification(params)` function that:
         a. Accepts: { userId: string, title: string, body: string, data: Record<string, string>, preferenceKey: keyof NotificationPreferences }
         b. Queries push_tokens table for user's expo_push_token
         c. Queries notification_preferences table — if preference for preferenceKey is false, skip sending (return early)
         d. If no preferences row exists, assume all enabled (default true)
         e. POSTs to `https://exp.host/--/api/v2/push/send` with:
            ```json
            {
              "to": expoPushToken,
              "sound": "default",
              "title": title,
              "body": body,
              "data": data,
              "channelId": "default"
            }
            ```
         f. Returns { sent: boolean, reason?: string }
       - `sendBulkPushNotifications(params)` function that:
         a. Accepts: { userIds: string[], title: string, body: string, data: Record<string, string>, preferenceKey: keyof NotificationPreferences }
         b. Queries push_tokens for all userIds
         c. Queries notification_preferences for all userIds, filters out those with preference disabled
         d. Batches into chunks of 100 (Expo API limit per request)
         e. POSTs each batch to Expo Push API
         f. Returns { sent: number, skipped: number }
       - Use createSupabaseAdmin() for DB queries (server-side, not user-scoped)
       - IMPORTANT: Use native fetch (not axios) — Next.js 15 has built-in fetch

    2. Create src/app/api/notifications/send/route.ts:
       - POST endpoint accepting: { userId: string, type: string, title: string, body: string, data: Record<string, string> }
       - Maps `type` to preferenceKey: "nearby_event" -> "nearby_events", "badge" -> "badge_unlocks", "tier_up" -> "tier_ups", "artist_message" -> "artist_messages", "friend_join" -> "friend_joins", "weekly_recap" -> "weekly_recap"
       - Calls sendPushNotification with appropriate preferenceKey
       - Returns 200 with { sent: boolean }
       - Protected by checking for an API secret header (X-Notification-Secret matching NOTIFICATION_SECRET env var) to prevent unauthorized sends
       - If no secret configured, allow calls from same origin (for internal use during dev)

    IMPORTANT: The Expo Push API is free and does not require Firebase setup for sending. Expo handles the FCM/APNs relay. No Firebase service account or API key needed on the server side.
