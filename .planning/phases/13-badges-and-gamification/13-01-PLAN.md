1. Create `src/lib/types/badges.ts` with these types:
   - `BadgeCategory = "discovery" | "attendance" | "exploration" | "streak" | "social"`
   - `BadgeRarity = "common" | "rare" | "epic" | "legendary"`
   - `BadgeId` = string literal union of all badge IDs (e.g., "trailblazer", "first-100", "first-10-verified", "regular", "devotee", etc.)
   - `BadgeDefinition = { id: BadgeId; name: string; description: string; category: BadgeCategory; rarity: BadgeRarity; icon: string (emoji); criteria: string (human-readable); }`
   - `EarnedBadge = { badge_id: BadgeId; fan_id: string; earned_at: string; }`
   - `BadgeWithDefinition = EarnedBadge & { definition: BadgeDefinition }`

2. Create `src/lib/badges/definitions.ts` with `BADGE_DEFINITIONS: Record<BadgeId, BadgeDefinition>`:

   **Discovery (BADGE-01):**
   - `trailblazer`: "Discovered 10 artists online" — rare
   - `first-100`: "Among first 100 fans on Decibel" — legendary (check fan.created_at order)
   - `first-10-verified`: "10 verified collections" — epic

   **Attendance (BADGE-02):**
   - `regular`: "Attended 5 shows" (5 verified collections) — common
   - `devotee`: "Attended 20 shows" — rare
   - `inner-circle-badge`: "Reached Inner Circle tier with any artist" (any fan_tiers row with current_tier=inner_circle) — epic
   - `venue-local`: "Visited same venue 5 times" — rare
   - `venue-legend`: "Visited same venue 20 times" — legendary

   **Exploration (BADGE-03):**
   - `genre-explorer`: "Collected artists across 5+ genres" — common
   - `city-hopper`: "Attended shows in 3+ cities" — rare
   - `night-owl`: "Collected at 3+ late-night venues" (shows after midnight based on event_date, or just any 3+ unique venues as fallback) — common
   - `scene-veteran`: "Member for 6+ months" — rare
   - `centurion`: "100 total collections (verified + discovered)" — legendary

   **Streak (BADGE-04):**
   - `on-fire`: "3-week attendance streak" — common
   - `unstoppable`: "8-week attendance streak" — rare
   - `year-round`: "26-week attendance streak (half a year)" — legendary

   **Social (BADGE-05):**
   - `tastemaker`: "Discovered 25 artists" — rare
   - `connector`: "Shared passport publicly" (has at least 1 share action — for now, award if fan has 10+ total collections as proxy until share tracking exists) — common

   Export `getBadgesByCategory(category: BadgeCategory): BadgeDefinition[]` helper.

3. Create `src/lib/badges/engine.ts`:
   - `interface FanBadgeData` — all data needed for evaluation: `{ fanId: string; fanCreatedAt: string; totalVerified: number; totalDiscovered: number; totalCollections: number; uniqueGenres: number; uniqueCities: number; uniqueVenues: number; maxVenueVisits: number; currentStreak: number; hasInnerCircle: boolean; fanRank: number | null; }` (fanRank = position among all fans by created_at, for first-100 badge)
   - `evaluateBadges(data: FanBadgeData): BadgeId[]` — pure function, checks each badge's criteria against data, returns list of earned badge IDs
   - `async function buildFanBadgeData(fanId: string, admin: SupabaseClient): Promise<FanBadgeData>` — queries collections, fan_tiers, fans tables to build the data object
   - `async function checkNewBadges(fanId: string, admin: SupabaseClient): Promise<BadgeId[]>` — calls buildFanBadgeData, evaluateBadges, diffs against existing fan_badges rows, inserts new ones, returns newly earned badge IDs

   The `evaluateBadges` function must be pure (no DB calls) so it's testable. All DB interaction goes through `buildFanBadgeData` and `checkNewBadges`.

   Use Supabase admin client for all queries. The fan_badges table should be: `fan_badges (id uuid PK default gen_random_uuid(), fan_id uuid FK, badge_id text, earned_at timestamptz default now(), UNIQUE(fan_id, badge_id))`.

   IMPORTANT: Create the fan_badges table via the admin client in the engine's init if it doesn't exist, OR document that the table must be created. Since we can't run DDL via Supabase JS client, include a SQL comment block at the top of engine.ts with the CREATE TABLE statement that needs to be run manually. The backfill script (Task 2) should also attempt to create it.
