  <action>
1. Create `src/lib/types/discovery.ts` with:
   - `ResolvedArtist`: { name, platform, platform_url, photo_url?, soundcloud_url?, ra_url?, instagram_handle?, genres? }
   - `DiscoverRequest`: { performer_id?: string, resolved_artist?: ResolvedArtist } (either existing or new)
   - `DiscoverResponse`: { success, collection_id, performer_id, performer_name, already_discovered }
   - `SupportedPlatform`: "spotify" | "soundcloud" | "ra" | "instagram" | "tiktok" | "youtube"
   - `LinkResolveResponse`: { resolved: boolean, artist?: ResolvedArtist, existing_performer?: { id, name, slug, photo_url }, error?: string }

2. Create `src/app/api/discover/resolve-link/route.ts` (POST endpoint):
   - Accepts `{ url: string }` body. Requires auth (use createSupabaseServer to verify user).
   - Parse the URL to detect platform (regex patterns for each supported platform).
   - Extract artist identifier from URL:
     - Spotify: `/artist/{id}` -> extract artist name from URL path or Open Graph (for now, extract from URL slug)
     - SoundCloud: `soundcloud.com/{username}` -> use the SoundCloud widget API (`api-widget.soundcloud.com/resolve?url=${url}&format=json&client_id=nIjtjiYnjkOhMyh5xrbqEW12DxeJVnic`) to get name, avatar_url, description
     - RA: `ra.co/dj/{slug}` -> extract slug as name (capitalize, replace hyphens)
     - Instagram: `instagram.com/{handle}` -> extract handle as name
     - TikTok: `tiktok.com/@{handle}` -> extract handle as name
     - YouTube: `youtube.com/@{handle}` or `/channel/{id}` -> extract handle/name from URL
   - After resolving, check if performer already exists in DB: query `performers` table by name (case-insensitive ILIKE) OR by matching soundcloud_url/ra_url/instagram_handle.
   - Return `LinkResolveResponse` — if performer found, return existing_performer data. If not, return resolved artist metadata for auto-creation.
   - Handle invalid/unsupported URLs gracefully with error message.

3. Create `src/app/api/discover/route.ts` (POST endpoint):
   - Accepts `DiscoverRequest` body. Requires auth.
   - Get fan from auth user email (same pattern as passport/page.tsx).
   - If `performer_id` provided (existing performer): insert collection with `{ fan_id, performer_id, capture_method: "online", verified: false }`. Handle unique constraint (already discovered).
   - If `resolved_artist` provided (new performer):
     a. Generate slug from name (lowercase, replace spaces with hyphens, strip special chars).
     b. Check slug uniqueness, append random 4 chars if collision.
     c. Insert into `performers` table: `{ name, slug, photo_url, soundcloud_url, ra_url, instagram_handle, city: null, genres: resolved_artist.genres || [], claimed: false }`.
     d. Then insert collection as above.
   - Return `DiscoverResponse`.
   - Use createSupabaseAdmin for DB writes (service role), createSupabaseServer for auth check.
  </action>
