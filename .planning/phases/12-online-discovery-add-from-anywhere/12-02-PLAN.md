<task type="auto">
  <name>Task 1: Spotify OAuth flow and top artists import API</name>
  <files>src/app/api/spotify/auth/route.ts, src/app/api/spotify/callback/route.ts, src/app/api/spotify/import/route.ts</files>
  <action>
1. Create `src/app/api/spotify/auth/route.ts` (GET):
   - Requires auth (createSupabaseServer to check user).
   - Build Spotify authorize URL with:
     - client_id from `process.env.SPOTIFY_CLIENT_ID`
     - redirect_uri: `${process.env.NEXT_PUBLIC_SITE_URL || 'https://decibel-swarn-singhs-projects.vercel.app'}/api/spotify/callback`
     - scope: `user-top-read` (minimum needed)
     - state: fan's user ID (for verification in callback)
     - response_type: code
   - Redirect to Spotify authorize URL.

2. Create `src/app/api/spotify/callback/route.ts` (GET):
   - Exchange `code` query param for access token via POST to `https://accounts.spotify.com/api/token` with client_id, client_secret, grant_type=authorization_code, code, redirect_uri.
   - Use Basic auth header (base64 of client_id:client_secret).
   - Store access_token in a secure httpOnly cookie `spotify_token` (expires in 1 hour matching Spotify token expiry).
   - Redirect to `/passport?spotify=connected`.
   - Handle errors: if no code or token exchange fails, redirect to `/passport?spotify=error`.

3. Create `src/app/api/spotify/import/route.ts` (POST):
   - Requires auth. Read `spotify_token` from cookies.
   - If no token, return 401 with `{ error: "Spotify not connected" }`.
   - Fetch top artists from Spotify API: GET `https://api.spotify.com/v1/me/top/artists?limit=20&time_range=medium_term` with Bearer token.
   - For each Spotify artist:
     a. Check if performer exists in DB by name (case-insensitive ILIKE).
     b. If exists: insert collection with `{ fan_id, performer_id, capture_method: "online", verified: false }`. Skip if already collected (handle 23505 unique violation).
     c. If not exists: create performer with `{ name, slug (generated), photo_url (from Spotify images[0].url), genres (from Spotify genres), claimed: false }`, then insert collection.
   - For each imported artist, also check if they have upcoming events in the `events` table (join events on performer_id, where event_date >= today). Flag these as `has_upcoming_show: true` with event details.
   - Return `{ imported: number, already_had: number, artists: Array<{ name, performer_id, photo_url, already_discovered, has_upcoming_show, next_show?: { venue_name, event_date } }> }`.
   - Delete the spotify_token cookie after import (one-time use).
   - Use createSupabaseAdmin for DB writes.
  </action>
