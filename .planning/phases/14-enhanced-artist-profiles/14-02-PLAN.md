  <name>Task 1: Journey state API endpoint</name>
  <files>src/app/api/artist/[slug]/journey/route.ts</files>
  <action>
Create GET /api/artist/[slug]/journey -- returns the logged-in fan's journey state with this artist.

1. Auth check via createSupabaseServer. If not logged in, return `{ authenticated: false, state: "none" }`
2. Look up performer by slug
3. Look up fan by user email
4. Query collections for (fan_id, performer_id):
   - No collection: state = "none"
   - Collection with verified=false only: state = "discovered"
   - Collection with verified=true: state = "collecting" or higher
5. Query fan_tiers for (fan_id, performer_id) to get scan_count and current_tier
6. Determine journey state:
   - "none": never interacted
   - "discovered": has online discovery but no verified collection
   - "collecting": has verified collection, tier is network or early_access
   - "devoted": tier is secret
   - "inner_circle": tier is inner_circle
7. Return JSON:
```json
{
  "authenticated": true,
  "state": "discovered" | "collecting" | "devoted" | "inner_circle" | "none",
  "scan_count": 0,
  "current_tier": "network" | null,
  "next_tier": "early_access" | null,
  "scans_to_next": 2
}
```

Use createSupabaseServer for auth and createSupabaseAdmin for data queries (fan email lookup).

**artist-actions.tsx (client component):**
"use client" component that receives `performerId`, `performerSlug`, `performerName`, and `nextShowVenue` (string | null) as props.

On mount, fetch GET /api/artist/{slug}/journey to get the fan's journey state.

Render based on state:

**If not authenticated (state="none", authenticated=false):**
- Show "Discover" button (gradient outline style) -> redirects to /auth/login?redirect=/artist/{slug}
- Show "Collect" button (solid gradient fill) -> redirects to /auth/login?redirect=/collect/{slug}

**If authenticated, state="none" (no relationship):**
- Show "Discover" button -> calls POST /api/discover with { performer_id } -> on success, show toast "Added to your passport!" via sonner, update local state to "discovered"
- Show "Collect" button -> if nextShowVenue, show "Next show at {venue}" tooltip/subtext, link to /collect/{slug}. If no next show, show "Collect at their next show" with muted style.

**If state="discovered":**
- Show "Discovered" badge (checkmark icon, teal outline) - not clickable
- Show "Collect in Person" button -> link to /collect/{slug} or show next show info
- Show journey indicator: "Discovered -- Collect at a show to level up"

**If state="collecting" or higher:**
- Show tier badge with current tier name and color (from TIER_COLORS)
- Show progress indicator: "X/Y scans to {next_tier}" as a small progress bar
- If inner_circle: show "Inner Circle" badge with gold/teal styling, no progress bar needed
- Show "See Next Show" link if upcoming event exists

**Journey state indicator (below the buttons):**
A horizontal stepper showing: Discover -> Collect -> Inner Circle
Highlight the current step. Use subtle connecting lines between steps. Decibel dark aesthetic.

**Integration into page.tsx:**
