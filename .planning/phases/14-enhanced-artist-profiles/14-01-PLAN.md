  <name>Task 1: Fan stats and similar artists API endpoints</name>
  <files>src/app/api/artist/[slug]/stats/route.ts, src/app/api/artist/[slug]/similar/route.ts</files>
  <action>
Create two public GET API endpoints (no auth required -- these are public artist profile data):

**GET /api/artist/[slug]/stats:**
1. Look up performer by slug
2. Query collections table for this performer_id:
   - Total collectors: count where verified=true
   - Total discoverers: count where verified=false (capture_method='online')
   - Total fans: sum of both
3. Query fan_tiers for this performer_id, group by current_tier to get tier breakdown (e.g., {network: 12, early_access: 5, secret: 2, inner_circle: 1})
4. Return JSON: `{ total_fans, collectors, discoverers, tier_breakdown: Record<string, number> }`

**GET /api/artist/[slug]/similar:**
1. Look up performer by slug, get their genres array
2. If no genres, return empty array
3. Query performers table using `.overlaps('genres', performerGenres)` to find artists sharing at least one genre
4. Exclude the current performer from results
5. Limit to 8 results, ordered by follower_count desc (most popular first)
6. Return JSON: `{ similar: Array<{ id, name, slug, photo_url, genres, city }> }`

Use createSupabaseServer for both endpoints (read-only, no admin needed).
