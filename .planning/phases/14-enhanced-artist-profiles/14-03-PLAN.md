  <action>
**POST /api/claim/request endpoint:**
1. Accept JSON body: `{ performer_id: string, email: string }`
2. Validate email format (basic regex)
3. Check performer exists and is not already claimed
4. Use Supabase Auth admin to send a magic link to the provided email:
   - Use `supabase.auth.admin.generateLink({ type: 'magiclink', email, options: { redirectTo: '{SITE_URL}/auth/callback?claim={performer_id}' } })` if available
   - OR use `supabase.auth.signInWithOtp({ email, options: { emailRedirectTo: '{SITE_URL}/auth/callback?claim={performer_id}' } })` via createSupabaseAdmin
   - The claim param in the redirect URL tells the callback handler to claim the performer after authentication
5. Return `{ success: true, message: "Check your email for a verification link" }`

Note: The existing /auth/callback route already handles redirect after auth. It checks for claimed performers. We need to ensure it handles the `claim` query param:
- After auth, if `claim` param exists with a performer_id, call the existing claim logic (set claimed=true, claimed_by=user.id) then redirect to /dashboard
- This modification to auth/callback/route.ts is part of this task

"use client" component that receives `performerId` and `performerName` as props. Only rendered when performer.claimed === false.
