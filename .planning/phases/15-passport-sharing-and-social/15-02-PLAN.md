**Migration SQL** (`supabase/migrations/003_social.sql`):
Create two tables (document as manual prerequisite like fan_badges -- Supabase JS client can't run DDL):

```sql
-- Fan follows (social graph)
create table if not exists fan_follows (
  id uuid primary key default gen_random_uuid(),
  follower_id uuid references fans(id) on delete cascade not null,
  following_id uuid references fans(id) on delete cascade not null,
  created_at timestamptz default now(),
  unique(follower_id, following_id)
);

create index if not exists idx_fan_follows_follower on fan_follows(follower_id);
create index if not exists idx_fan_follows_following on fan_follows(following_id);

-- Fan privacy settings
create table if not exists fan_privacy (
  fan_id uuid primary key references fans(id) on delete cascade,
  visibility text not null default 'public' check (visibility in ('public', 'private', 'mutual')),
  updated_at timestamptz default now()
);
```

**Types** (`src/lib/types/social.ts`):
```typescript
export type PrivacySetting = 'public' | 'private' | 'mutual';
export type FollowStatus = 'not_following' | 'following' | 'mutual';

export interface SocialCounts {
  followers: number;
  following: number;
}

export interface FollowResponse {
  status: FollowStatus;
  counts: SocialCounts;
}
```

**Follow endpoint** (`POST /api/social/follow`):
- Body: `{ targetFanId: string, action: 'follow' | 'unfollow' }`
- Auth required (get user from session, look up fan by email)
- Follow: insert into fan_follows (follower_id=current fan, following_id=target)
- Unfollow: delete from fan_follows matching both IDs
- Return updated FollowStatus and SocialCounts
- Prevent self-follow (return 400)
- Use admin client for DB operations (per project pattern)

**Followers endpoint** (`GET /api/social/followers?fanId={id}`):
- Returns list of followers with name, slug, created_at
- No auth required (public data, respects privacy via separate check)

**Following endpoint** (`GET /api/social/following?fanId={id}`):
- Returns list of fans this fan follows with name, slug, created_at

**Privacy endpoint** (`GET/PUT /api/social/privacy`):
- GET: return current user's privacy setting (auth required)
- PUT: update privacy setting, body `{ visibility: PrivacySetting }`
- Auth required for both
- Default to 'public' if no row exists
